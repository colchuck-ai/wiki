# CR010 - Read-only freshness interface: index_status and recency_status

Gave Lint a **read-only** way to check index and recency freshness. Added two dry-run interfaces — `C005.index_status(scope?) → stale_paths[]` and `C007.recency_status(scope?) → unrecorded_paths[]` — and rewired every reference to Lint's freshness check to name them, in place of the prior spec that had Lint depend on `C005.reindex` idempotence (a *mutation*). This discharges the read-only-freshness concern [ADR016](../engineering/drs/ADR016-survey-lint-aggregation-ownership.md) deferred and makes C011's "a Lint pass mutates nothing" contract buildable rather than self-contradictory.

## Change

Before: the engineering README specced Lint's index/recency freshness check as "`C005.reindex` idempotence, `C007.recency`." But `reindex` is C005's *only* corpus mutation — it writes the corrected `index.md` when the index is stale. A read-only Lint therefore had no way to run its own specced check: it would have to either call `reindex` (violating "Lint mutates nothing") or re-derive the index itself (violating one-owner-of-index-derivation). C011 - Curation Operations — added later — was already written to read "the read-only index/recency freshness signal C005 and C007 expose" and explicitly deferred the interface ("The exact read-only freshness interface is C005's and C007's to provide"), and ADR016 recorded the same deferral as "the separately-tracked read-only-freshness concern." No such read-only interface existed on either component: C005 exposed only `reindex` (mutating) plus `inbound_links` / `dangling_links` / `graph`; C007 exposed per-concept `recency` but no scope-wide freshness sweep.

After:

- **`C005.index_status(scope?) → stale_paths[]`** (new, read-only) — returns the paths whose on-disk `index.md` differs from what `reindex` would regenerate, and writes nothing. It is `reindex`'s derivation minus the write, so C005 stays the one owner of index derivation while a read-only caller detects out-of-band index drift without mutating.
- **`C007.recency_status(scope?) → unrecorded_paths[]`** (new, read-only) — returns the concepts in scope carrying no recorded `timestamp` (authored before the tooling, or via a path that bypassed the verb contract), and writes nothing. Its reach is deliberately narrow — it detects the *absence* of a recording, not a concept hand-edited after being recorded — because C007 records call-time, not content-derived time, and so holds no ground truth for judging a present-but-stale timestamp.
- **C011 - Curation Operations** — `lint` now reads `C005.index_status` and `C007.recency_status` by name (interface, relationships, routing table, and the deferral note that pinned "no mutation"); the interface it had deferred is now supplied.
- **Engineering README, Lint operation** — the broken "(`C005.reindex` idempotence, `C007.recency`)" is replaced with the two read-only dry-runs, and the two residual-drift cases it already named are attributed to them: "authored before the tooling" → `recency_status`, "hand-edited outside the verbs" → `index_status`.
- **ADR016** — the consequence that deferred the interface now points forward to this change as its resolution.

No requirement, risk, or Requirement-Component Map row changes: this fills interfaces on components (C005, C007) already mapped to their outcomes, and touches no product element.

## Rationale

[ADR011](../engineering/drs/ADR011-concept-verb-surface.md) makes index and recency drift "eliminated by construction" — each write verb regenerates the index and records recency as an internal effect, so no forgotten terminal call can drift them. But ADR011 itself notes one residual: a corpus changed *outside the verb surface* (a raw edit on disk) still drifts, "a stewardship gap to catch via Lint." Catching it is precisely Lint's job — and Lint is read-only by contract, so the check it runs must be read-only too. The prior spec named a mutation for a read-only check, which cannot be built as written; C011 then encoded the correct intent ("read-only freshness signal, never `reindex`") but against an interface that did not exist. This change supplies that interface.

The two dry-runs split cleanly along where a ground truth exists. The index has one — the concepts on disk — so `index_status` is well-defined: recompute and diff, don't write. Recency does not: C007 sets `timestamp` to the recording moment by design, not to any content-derived time, so the only recency drift it can honestly detect is the *absence* of a recording (`recency_status`), not staleness after one. Stating that limit is deliberate: a body-only edit that changes neither the index entry nor a timestamp's presence is undetectable by any read-only check, the irreducible cost of recording by intent rather than watching the corpus. Alternatives to `index_status` — Lint re-deriving the index (duplicates C005's derivation, breaks one-owner) or calling `reindex` and reading an empty `updated_paths` as "fresh" (a mutation on a read path) — were rejected for the dry-run. No new decision-with-alternatives was opened: ADR016 already recorded the deferral and anticipated "a future tightening … without a contract change here," which is exactly this.

## Affects

- **C005 - Index & Navigation** — gains `index_status(scope?) → stale_paths[]` (read-only dry-run); the out-of-band-drift edge case, the C011 relationship, a no-mutation and a freshness-fidelity success criterion, and a Notes paragraph on the dry-run and its rejected alternatives all name it; See Also gains this CR.
- **C007 - Currency Tracking** — gains `recency_status(scope?) → unrecorded_paths[]` (read-only sweep of unrecorded concepts); the never-recorded edge case, a no-mutation and a freshness-sweep success criterion, and the Notes line on the Lint freshness signal all name it; See Also gains this CR.
- **C011 - Curation Operations** — its `lint` interface, C004/C005 relationship, routing table, and the note that pinned "no mutation" now name `C005.index_status` and `C007.recency_status`; the deferred interface is discharged. See Also gains this CR.
- **Wiki Curation Architecture** — the Lint operation names the two read-only dry-runs in place of `reindex` idempotence; See Also gains this CR.
- **ADR016 - Survey/Lint aggregation ownership** — the consequence that deferred the read-only-freshness interface now points forward to this change as its resolution.
- **ADR011 - Concept verb surface** — unchanged; its "drift eliminated by construction, residual caught via Lint" claim is now buildable, since the Lint catch has a read-only interface.
